## 1. Executive Summary

### Tujuan Project

`claude-sync` adalah CLI Python untuk **mensinkronkan data session Claude Code** antar device menggunakan **Git** sebagai media transport. Secara spesifik, tool ini menyalin folder konfigurasi Claude Code (`~/.claude/`) yang berisi sessions, tasks, plans, session-env, projects, memory, dan beberapa file individual (history.jsonl, CLAUDE.md, settings.json) dari satu mesin ke mesin lain.

### Kemampuan Project Saat Ini

| Command | Fungsi | Status |
|---------|--------|--------|
| `init` | Membuat `.claude-sync/` + `manifest.json` di project root | ✅ |
| `status` | Cek inisialisasi project + deteksi instalasi Claude Code | ✅ |
| `inspect` | Tampilkan struktur & statistik `~/.claude/` (read-only) | ✅ |
| `export` | Salin `~/.claude/` → `.claude-sync/data/` | ✅ |
| `import` | Salin `.claude-sync/data/` → `~/.claude/` (dengan backup otomatis) | ✅ |
| `push` | Export + git add/commit/push | ✅ |
| `pull` | git fetch/pull + import | ✅ |
| `trace` | Snapshot & diff filesystem `.claude/` | ✅ |

- **Platform:** Linux, macOS (via Darwin resolver), Windows, WSL
- **Test coverage:** 11 test files, ~60+ test cases
- **Tech stack:** Python 3.12+, Typer 0.12+, Rich 13.7+, pytest 8+, ruff 0.5+

### Batasan Project Saat Ini

1. **Tidak project-based** — data export/import bersifat **global** (`~/.claude/` penuh), bukan per-project atau per-session
2. **No encryption** — data disimpan sebagai plain file di Git repo
3. **No path mapping** — assumes `~/.claude/` ada di lokasi sama di semua device
4. **No conflict resolution** — "wipe and rewrite" semantics; data lama di-override tanpa merge
5. **No incremental sync** — setiap export meng-wipe dan copy ulang seluruh directory
6. **No filtering** — seluruh `EXPORT_SUBDIRS` dan `EXPORT_FILES` disalin semuanya
7. **No auto-restore awareness** — tidak tahu session mana yang "aktif" atau "paling baru"

| `utils/config.py` | Project root resolution |

### Class yang Terlibat

- **`ClaudeExporter`** (`utils/exporter.py`) — Main export engine
  - Constructor: `(claude_path: Path, project_root: Path, subdirs: Iterable[str])`
  - Method: `export() → ExportReport`
  - Property: `data_root` → `project_root/.claude-sync/data`

- **`ExportReport`** (`utils/exporter.py`) — Dataclass result
  - Fields: `claude_path`, `data_root`, `copied` (dict), `skipped` (tuple), `file_count`, `copied_files`, `skipped_files`

### Fungsi yang Terlibat

| Fungsi | Lokasi | Peran |
|--------|--------|-------|
| `ClaudeExporter.export()` | `utils/exporter.py` | Main export method |
| `_count_files(root)` | `utils/exporter.py` | Count files recursively for report |

### Data yang Dihasilkan

Export menyalin 2 jenis data:

**A. Subdirectories (copytree — recursive):**

```
EXPORT_SUBDIRS = ("sessions", "tasks", "plans", "session-env", "projects", "memory")
```

Masing-masing disalin dari `<claude_path>/<subdir>` → `<data_root>/<subdir>`.

**B. Individual files (copy2 — single file):**

```
EXPORT_FILES = ("history.jsonl", "CLAUDE.md", "settings.json")
```

Disalin dari `<claude_path>/<file>` → `<data_root>/<file>`.

### Lokasi Output

```
<project_root>/
└── .claude-sync/
    ├── manifest.json          ← not touched by export
    └── data/                  ← wiped & rewritten every export
        ├── sessions/
        ├── tasks/
        ├── plans/
        ├── session-env/
        ├── projects/
        ├── memory/
        ├── history.jsonl
        ├── CLAUDE.md
        └── settings.json
```

### Diagram Alur Export

```
claude-sync export
    │
    ├─ Check: project initialized? ──No──→ exit(1) "not initialized"
    │                                          Yes
    ├─ Locate source: claude_locator ──None──→ exit(1) "Not Found"
    │              │
    │         Yes (Path)
    ├─ Wipe .claude-sync/data/ (shutil.rmtree)
    ├─ For each subdir in EXPORT_SUBDIRS:
    │   ├─ If source exists: shutil.copytree → data/<subdir>
    │   └─ Else: skip (recorded in skipped)
    ├─ For each file in EXPORT_FILES:
    │   ├─ If source exists: shutil.copy2 → data/<file>
    │   └─ Else: skip (recorded in skipped_files)
    ├─ Build ExportReport
    └─ Print summary table to console
```

## 6. Import Architecture

### File yang Terlibat

| File | Peran |
|------|-------|
| `commands/import_cmd.py` | CLI command handler — orchestrates import |

### Class yang Terlibat

- **`ClaudeImporter`** (`utils/importer.py`) — Main import engine
  - Constructor: `(claude_path: Path, project_root: Path, subdirs: Iterable[str])`
  - Method: `import_data(backup: bool = True) → ImportReport`
  - Property: `data_root` → `project_root/.claude-sync/data`

- **`ImportReport`** (`utils/importer.py`) — Dataclass result
  - Fields: `claude_path`, `data_root`, `backup_path`, `restored` (dict), `skipped`, `file_count`, `backup_existed`, `restored_files`, `skipped_files`

### Fungsi yang Terlibat

| Fungsi | Lokasi | Peran |
|--------|--------|-------|
| `ClaudeImporter.import_data()` | `utils/importer.py` | Main import method |
| `ClaudeImporter._make_backup_path()` | `utils/importer.py` | Build `backup-YYYYMMDD-HHMMSS[-N]` path |
| `_count_files(root)` | `utils/importer.py` | Mirrors exporter's helper |

### Proses Backup

1. **Backup path construction:** `<parent>/.claude.backup-YYYYMMDD-HHMMSS`
2. **Collision handling:** If backup already exists, append `-1`, `-2`, etc.
3. **Backup method:** `shutil.move()` — atomic rename on same filesystem
4. **No backup:** `--no-backup` flag wipes existing `.claude/` before importing

### Diagram Alur Import

```
claude-sync import
    │
    ├─ Check: project initialized? ──No──→ exit(1)
    │                                          Yes
    ├─ Locate destination: claude_locator ──None──→ exit(1)
    │              │
    │         Yes (Path)
    ├─ Check: .claude-sync/data/ exists? ──No──→ exit(1) "nothing to import"
    │              │
    │         Yes
    ├─ If backup=True AND .claude/ exists:
    │   ├─ Backup: shutil.move(.claude, .claude.backup-TIMESTAMP)
    │   └─ Record backup_path
    ├─ Else if backup=False AND .claude/ exists:
    │   └─ shutil.rmtree(.claude)  (opt-in destructive)
    ├─ mkdir .claude (parents=True, exist_ok=True)
    ├─ For each subdir in EXPORT_SUBDIRS:
    │   ├─ If data/<subdir> exists: shutil.copytree → .claude/<subdir>
    │   └─ Else: skip
    ├─ For each file in EXPORT_FILES:
    │   ├─ If data/<file> exists: shutil.copy2 → .claude/<file>
    │   └─ Else: skip
    ├─ Build ImportReport
    └─ Print summary table to console
```

## 7. Git Integration

### File yang Terlibat

| File | Peran |
|------|-------|
| `utils/git_sync.py` | Core: `GitSync` class, `GitError`, `GitResult` |
| `commands/push.py` | Orchestration: export → git operations |
| `commands/pull.py` | Orchestration: git operations → import |

### Class yang Terlibat

- **`GitSync`** (`utils/git_sync.py`) — Git wrapper
  - Constructor: `(project_root, remote="origin", branch="main", sync_dir=None)`
  - Methods: `is_git_available()`, `is_repo()`, `has_remote()`, `has_uncommitted_changes()`
  - Mutating: `init_repo()`, `add(paths)`, `commit(message)`, `push()`, `pull()`, `fetch()`
  - Property: `sync_dir` defaults to `project_root/.claude-sync`

- **`GitError`** — Raised on non-zero exit or missing git binary
- **`GitResult`** — Frozen dataclass: `returncode`, `stdout`, `stderr`, `command`

### Push Flow

```
claude-sync push
    │
    ├─ Check: initialized? ──No──→ exit(1)
    │
    ├─ If not --no-export:
    │   └─ export()  (wipe data, copy from ~/.claude)
    │
    ├─ git.is_git_available()? ──No──→ exit(1)
    │
    ├─ git.is_repo()? ──No──
    │   ├─ --no-init? → exit(1)
    │   └─ init_repo()  (git init --initial-branch <branch>)
    │
    ├─ git.has_remote(remote)? ──No──→ exit(1) "no remote"
    │
    ├─ skip_empty AND no changes? → exit(0) (informational)
    │
    ├─ git.add()  (stage .claude-sync/)
    ├─ git.commit(msg)  (timestamped message)
    └─ git.push()  (push -u origin main)
```

### Pull Flow

```
claude-sync pull
    │
    ├─ Check: initialized? ──No──→ exit(1)
    │
    ├─ git.is_git_available()? ──No──→ exit(1)
    │
    ├─ git.is_repo()? ──No──
    │   ├─ --no-init? → exit(1)
    │   └─ init_repo()
    │
    ├─ git.has_remote(remote)? ──No──→ exit(1)
    │
    ├─ git.fetch()
    └─ git.pull()  (pull --ff-only origin main)
        │
        ├─ If not --no-import:
        │   └─ import()  (backup → wipe → restore)
        │
        └─ Print success message
```

### Remote Detection

```
git.has_remote()
  → git rev-parse --abbrev-ref HEAD  (check current branch)
  → git config --get remote.<name>.url  (check if remote exists)
```

- **Default:** `Path.cwd()` — captured at function-call time (not import time), for test compatibility
- **Override:** `--project-root` / `-p` CLI option
- **Resolution:** `Path.resolve()` via `git_sync._resolve_root()`

## 11. Technical Debt

### HIGH Severity

## 12. Current Sync Limitation

### Mengapa Session Claude Code Tidak Dapat Dipindahkan Sempurna Antar Device

**Bukti dari source code:**

1. **Tidak ada sesi-per-session filtering**
   - `EXPORT_SUBDIRS` di `utils/exporter.py:23` menyalin **seluruh** `sessions/` directory.
   - Tidak ada mekanisme untuk memilih session mana yang akan dipindahkan.
   - Tidak ada metadata session (status, last_used, project association) yang diekspor.

2. **Tidak ada path mapping**
   - `ClaudeLocator` (`utils/claude_locator.py`) hanya mengembalikan satu path: `$HOME/.claude`.
   - Jika Device A menggunakan `~/.claude` dan Device B menggunakan `/opt/claude/.claude`, tidak ada auto-mapping.
   - `--claude-path` harus di-set manual.

3. **File system-level copy, bukan semantic sync**
   - `shutil.copytree` (`utils/exporter.py:113`) menyalin file secara literal.
   - Tidak ada parsing JSON, tidak ada merge, tidak ada conflict detection.
   - Session files di `sessions/` adalah JSON yang mungkin berisi device-specific paths.

4. **Wipe-and-rewrite = data loss potential**
   - `importer.py:114`: `shutil.move(.claude, .claude.backup-TIMESTAMP)` — data lokal di-device B **hilang** jika tidak di-backup.
   - Tidak ada warning sebelum overwrite.

5. **No session lifecycle awareness**
   - `ClaudeInspector.TRACKED_SUBDIRS` (`utils/inspector.py:14`) hanya menghitung entry count, tidak membaca isi sesi.
   - Tidak ada cara untuk mengetahui session mana yang "open", "closed", atau "running".

6. **Cross-platform path inconsistency**
   - `claude_locator.py:57-59`: Linux dan macOS menggunakan `$HOME`, Windows menggunakan `$USERPROFILE`. Tidak ada bridging antar platform.
   - Jika data di-export dari Linux dan di-import di Windows, semua path absolut di session files akan salah.

7. **Git repo per project, bukan per session**
   - `git_sync.py:68-69`: `sync_dir` default ke `.claude-sync/` di project root.
   - Artinya, satu project = satu Git repo = satu set data Claude Code.
   - Tidak ada cara untuk memindahkan session dari Project A ke Project B.

## 14. Refactor Recommendation

### Phase 1 — Foundation (Safe, Low Risk)

**Tujuan:** Persiapan infrastruktur tanpa mengubah behavior eksisting.

1. **Extract shared utilities**
   - Pindahkan `_resolve_root()` ke `utils/config.py`
   - Pindahkan `_count_files()` ke `utils/_shared.py`
   - Replace semua duplicate instances

2. **Add `.gitignore` generation**
   - `init.py` generate `.gitignore` entry untuk `.claude-sync/` jika belum ada

3. **Add `__all__` exports**
   - Tambahkan `__all__` di semua `utils/*.py` untuk eksplisit API surface

4. **Add encryption constants**
   - Tambahkan `ENCRYPTION_ENABLED = False` (default) sebagai feature flag

**Files affected:** `config.py`, `exporter.py`, `importer.py`, `push.py`, `pull.py`, `init.py`

---

### Phase 2 — Project Context (Medium Risk)

**Tujuan:** Project root menjadi konteks utama, bukan global `~/.claude/`.

1. **Extend `manifest.json` schema**
   ```json
   {
     "project_name": "string",
     "version": 1,
     "sync_target": "string",       // new: which claude path to sync
     "included_subdirs": ["list"],  // new: override EXPORT_SUBDIRS
     "included_files": ["list"],    // new: override EXPORT_FILES
     "path_mapping": {}             // new: source→dest path mappings
   }
   ```

2. **Update `ClaudeExporter` to accept project context**
   - Read manifest for `included_subdirs`, `included_files`, `sync_target`
   - Allow per-project filtering

3. **Update `ClaudeImporter` similarly**
   - Read manifest for sync target override
   - Apply path mapping during import

4. **Update `ClaudeLocator` to support project-specific paths**
   - New method: `find_claude_path_for_project(root)`

5. **Update `git_sync.py` to detect git remote from project root**
   - Not just from `.claude-sync/` dir

**Files affected:** `config.py`, `exporter.py`, `importer.py`, `claude_locator.py`, `git_sync.py`, `commands/init.py`, `commands/export.py`, `commands/import_cmd.py`

---

### Phase 3 — Cross-Platform & Session Awareness (Higher Risk)

**Tujuan:** Support cross-device dengan path mapping dan session granularity.

1. **Implement path mapping system**
   - New module: `utils/path_mapper.py`
   - Read mapping dari manifest: `{"/home/user/project": "/mnt/c/projects/project"}`
   - Apply during export (encode paths) and import (decode paths)

2. **Implement session-level export**
   - New command option: `--session <id>` di `export`
   - New command option: `--session <id>` di `import`
   - Parse session JSON metadata untuk filtering

3. **Add conflict detection**
   - Before import: scan both source and target
   - Report conflicts, ask user to resolve
   - Option: `--force` untuk override

4. **Add timestamp-aware session restore**
   - `import` prefer session dengan `last_used` paling baru
   - Not just wipe-and-copy

**Files affected:** New: `utils/path_mapper.py`, `utils/session_parser.py`; Modified: `exporter.py`, `importer.py`, `commands/export.py`, `commands/import_cmd.py`

**Tujuan:** Production-ready dengan enkripsi dan better packaging.

1. **Duplicate code** — 7x `_resolve_root()`, 2x `_count_files()`
